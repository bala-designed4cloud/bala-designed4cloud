WebSite : Cloud Landing Zone Design

Created by Kanesamoorthy Balakumar, last modified on Jan 30, 2022

Cloud Landing Zone has become very popular and initial introduction of reference implementation by AWS, which was called AWS Landing Zone solution for sure uplifted the popularity.

The Cloud Landing Zone though looks like an architecture pattern, the implementation in AWS and Azure and going forward from other Cloud provider as well, are quite different on how it’s implement, but still keeping basic capability of the landing zone quite close across all cloud providers.

Let’s ask some questions to clarify why you need such a landing zone in your organisation.

  1. How can I get access to a cloud tenant, account, subscription, project or whatever the smaller environment using my organisation’s credentials and comply to all of my organisations policies and standards?

  2. How can I create a cloud resources such as a VM, storage instance and deploy applications or upload my organisations data in a secure manager to comply with my organisation’s policy?

  3. How can I access my cloud resources from my devices to manage and access applications?

  4. How can I keep all my resources current across all my cloud environments?

  5. How can I monitor my applications for security and technical events and remediate soon as possible?

  6. How can I handle DR scenarios, how do I schedule backups, how to restore them and how to handle site to site DR?

  7. How can I do all these across 100s of cloud environments?

These are some of the questions to be asked before you go into Cloud Landing Zone design phase, as based on the answers the design cloud start from really simple to extremely complex, especially if you are going to have 100s of cloud environments, it’s almost given that you need a Cloud Landing Zone type solution to consistently to manage all of your cloud environments.

We have extensive experience in designing, building and managing Cloud Landing Zone at scale across multiple customers and across different cloud providers.
