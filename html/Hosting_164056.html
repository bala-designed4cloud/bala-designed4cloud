WebSite : Hosting

Created by Kanesamoorthy Balakumar, last modified on Jan 12, 2020

Introduction

Design Intents

  • Use always-available storage to keep the static content.

  • Minimise the compute requirement

  • Keep compute instance always stateless

Design Decisions

  • Use cloud storage to keep the static content

  • Use a small compute instance

  • Make static content in cloud storage accessible in compute instance

  • Use Apache HTTP server to publish the static content

Implementation

  • Mount cloud storage using fuse filesystem in compute instance

  • Keep the cloud storage content synched to compute local volume

    • cat /var/www/bin/storage_sync

      • gsutil rsync /var/www-01/html /var/www/html

      • gsutil rsync /var/www-01/html/js /var/www/html/js

      • gsutil rsync /var/www-01/html/css /var/www/html/css
